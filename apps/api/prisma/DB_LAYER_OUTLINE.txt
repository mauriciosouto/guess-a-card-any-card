Prompt 2 — repository / service layer map (BLUEPRINT_PRO.md §28.2)
================================================================

Card catalog (runtime; @flesh-and-blood/cards)
----------------------------------------------
Services:
  src/server/services/card-catalog-service.ts
    - Loads filtered in-memory catalog at API startup
    - getAllSets() / set lists for lobby
    - getRandomCard(selectedFabSets) for new games
    - searchPlayableCatalogCardNames() for autocomplete (HTTP)

Reveal / snapshot resolution
-----------------------------
  src/lib/game-card-resolution.ts
    - newRevealSeed()
    - resolveGameCard(game) — reads DB snapshot fields only (no joins)
  src/lib/reveal-profile.ts — plan step counts from seed + kind + template
  src/lib/fab-reveal-profile.ts — map official FAB Card → reveal profile
  src/lib/card-art-url.ts — hero image URLs for catalog cards

Single-player (BLUEPRINT_PRO.md §28.3)
--------------------------------------
Pure domain:
  src/lib/game/single-player-logic.ts
    - resolveSinglePlayerGuess()   — win / lose / advance step
    - singlePlayerAttemptCounts() — UI used / remaining

Services & HTTP:
  src/server/services/single-player-service.ts
    - startSinglePlayerGame(), getSinglePlayerGamePublic(), submitSinglePlayerGuess()
    - optional X-User-Id (registered → stats); else X-Guest-Id
  src/server/single-player-routes.ts  — /api/single/sets | POST /games | GET/POST .../guess

Stats & profile aggregates
--------------------------
Repositories:
  src/server/repositories/stats-repository.ts
    - ensureUserStatRow(userId)
    - applyGameOutcomeToUserStats(outcome)     — UserStat counters + bests
    - mergeUserCardStatsAfterGame(args)        — UserCardStat per catalog cardId

Services:
  src/server/services/stats-service.ts
    - recordRegisteredUserGameOutcome() — thin wrapper for game completion hooks

Multiplayer / COOP / COMPETITIVE persistence
-------------------------------------------
  src/server/services/coop-service.ts
  src/server/services/competitive-service.ts
  (Prisma directly; future: extract repositories if file size warrants)

Deterministic reveal overlays (client + server)
-----------------------------------------------
  packages/shared/src/reveal/*
  apps/web: src/components/game/StepCardPreview.tsx — seed + step + imageUrl
  apps/api: src/lib/puzzle/* — zone/plan math (naming legacy; not DB “puzzles”)

  HTTP APIs expose snapshot fields: cardImageUrl, revealSeed, revealCardKind,
  cardTemplateKey, plus cardId / cardName (terminal rules per mode).

Conventions
-----------
  - Guests: pass hostGuestId / guestId; registered users use hostUserId / userId.
  - Do not run prisma migrate reset on production without intent — see MIGRATION_STRATEGY.txt.
